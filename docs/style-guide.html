01 — Foundation

Colours

All colours are defined as CSS custom properties in globals.css. Never use raw hex values in component CSS — always reference the variable.

Primary
--primary #255b48

Buttons, project cards, stat cards, active states, brand accents. The dominant action colour.

Background
--background #f3eddf

Page background. The warm cream that defines the site's character. Never use pure white for backgrounds.

Surface
--surface #faf7f0

Card backgrounds. A step lighter than the page background, stays warm. Defined locally in analytics CSS — add to globals if used elsewhere.

Black
--black #191d1b

Body text, headings, primary content. Slightly warm (not pure black) — matches the cream background.

Grey
--grey #606060

Subheadings, captions, secondary text, table column headers, metadata.

White
--white #ffffff

Text on dark (primary) backgrounds. Avoid for page/card backgrounds — use --background or --surface instead.

Error
--error #a83c23

Form validation errors, destructive actions. Not currently used but reserved.

Live Green
(hardcoded) #22c55e

Live indicator dot only. One-off — no variable needed. Do not use for other purposes.

06 — Layout

Breakpoints

All breakpoints use max-width. Think mobile-last — desktop styles are the default, overrides apply downward.

≤1700px Wide Desktop Project card inner container shrinks to reduce image whitespace
≤1300px Desktop Project grid collapses from 3-col to 2-col
≤1024px Tablet (large) Analytics stat row changes from 5-col to 3-col
≤900px Tablet Analytics bento collapses to single column
≤768px Tablet (small) Project grid → 1-col; side margins reduce to 20px; stat row → 2-col
≤425px Mobile 16px side padding on all pages. Font sizes reduce. Project card radius → 16px. Stat cards: 5th card spans full width.

07 — Rule

Box-in-Box Radius

When nesting a rounded element inside another rounded element, the inner radius must be reduced by the gap between them to keep the visual rhythm consistent. This prevents corners that look too sharp or too round relative to the outer container.

Formula

Outer R = Inner R + Padding

Rearranged: Inner R = Outer R − Padding

Where Padding is the gap between the outer element's inner edge and the inner element's edge.
If Padding ≥ Outer R, the corners never overlap — the inner element's corners are in the flat zone, so Inner R = 0 (no rounding needed).

Project card (formula applies)

Outer 28px · Padding 20px · Inner 8px — all breakpoints

Medium gap (formula applies)

Outer 30px · Gap 25px · Inner 5px

Small gap (overflow: hidden)

Outer 16px · Gap 12px · Inner 4px + overflow:hidden on outer

Project card values (applied — all breakpoints)

Element Outer R Padding Inner R Note
Green project card → image 28px 20px 8px 28 − 20 = 8 · consistent at all screen sizes

Padding is applied via padding: 20px on the card, not calc(100% - Xpx) — this keeps the formula constant and the code simple. overflow: hidden on the card clips any edge overlap.

08 — Guidelines

Do & Don't

Design decisions that keep the site feeling cohesive.

Do
  • Use --background (#f3eddf) for page backgrounds
  • Use --surface (#faf7f0) for cards and elevated UI
  • Keep 16px side padding on mobile across all pages
  • Use CSS grid with gap for spacing — avoid margin-bottom between siblings
  • Use min-width: 0 on grid children to prevent content blowout
  • Apply text-overflow: ellipsis to long strings in tables
Don't
  • Use pure white (#fff) for page or card backgrounds
  • Use raw hex values in component CSS — always use variables
  • Use margin: 0 on containers at mobile (removes all breathing room)
  • Add box-shadow to cards — the border + background contrast is enough
  • Expose Supabase keys with NEXT_PUBLIC_ prefix — server-only only
  • Track IP addresses or personal data — use localStorage session IDs instead
Analytics Page Conventions
  • Card titles: 16px / 600 weight, --black
  • Use bento grid layout with gap: 20px
  • Dividers between list rows: 1px #e5dfd3
  • Bar track: #e5dfd3 — bar fill: --primary
  • Table headers: sticky, uppercase, 12px, --grey
Project Page Conventions
  • Max-width 728px, centered with margin: 0 auto
  • Title as a pill badge in --primary
  • Article body: 18px / 400, line-height 27px
  • Section headings: 22px h2, 18px h3
  • Images full-width within container
